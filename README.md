# 边缘计算与物联网融合的智能窗帘机器人装置

## 项目简介

本仓库保存智能窗帘机器人装置当前公开开发基线，覆盖嵌入式执行、边缘控制原型、移动端交互原型、电路设计和机械结构资料。项目目标是逐步形成可解释、可联调、可复核的软硬件装置，而不是声明已有可直接部署的成品。

## 当前包含内容

- STM32 嵌入式固件：电机控制、串口收发、RTC、传感器与低功耗相关工程。
- Python 控制程序：串口/GPIO 控制、自动控制和调试入口的现有实现。
- 微信小程序原型：基于 BLE 的早期控制页面与通信逻辑。
- PCB/EDA 资料：Altium 与立创 EDA 相关设计文件。
- SolidWorks/STL 机械结构资料：结构源文件与打印文件。
- 项目文档：串口协议说明、BOM 初稿、许可、贡献说明与 GitHub 协作模板。

## 能力边界

### 已核实

- STM32 固件已基础支持 `STOP` 与 `STATUS`，当前最小状态返回形如 `STATE:<state>;CAL:0`。
- `docs/串口协议草案.md` 已记录现有命令边界和目标协议方向。
- `docs/bom.md` 已提供基于公开设计资料的硬件物料核对初稿。

### 尚未闭环

- `CAL:0` 是当前基础占位返回，不代表校准状态或校准数据持久化已经完成。
- Python 串口客户端尚未完成 `STOP` / `STATUS` 发送、状态查询等待与解析接入；该能力属于下一开发任务。
- 微信小程序仍属于旧命令/BLE 控制原型，尚未对齐统一协议。
- Python、STM32、小程序、硬件实物之间的完整跨端控制闭环和整机验收尚未完成。
- BOM 是核对初稿，不代表器件已经采购、装配或通过实物验证。

## 开发入口

### STM32 固件

工程位置：

```text
01_软件/STM32嵌入式固件/My_curtain_robotV2_VSCode/
```

仓库包含 CMake/VS Code 相关工程配置。构建和烧录是否可用仍需按实际工具链、目标板和连接环境验证。

### Python 控制程序

程序位置：

```text
01_软件/Python控制程序/
```

该目录已有主程序、串口通信和控制模块。目前串口侧只可确认现有 `ON`、`OFF`、`C` 发送路径；下一开发任务是接入 STM32 已有的 `STOP` / `STATUS` 最小能力。

### 微信小程序

工程位置：

```text
01_软件/微信小程序/My_Curtain_robot_Wechart/
```

该工程当前用于 BLE 交互原型验证；其中命令仍需与后续统一协议对齐。

## 开发优先级

当前下一优先功能任务是让 Python 串口客户端接入 STM32 已有的 `STOP` / `STATUS` 最小能力，包括发送停止命令、发起状态查询、等待并解析最小状态返回及处理超时结果。

## 使用与验证注意事项

- 本仓库不能作为量产硬件、完成整机或已通过安全验证的声明。
- 机械运动、电机负载、电源与电池、安装可靠性和异常停止能力均需要实物验证。
- 进行跨端开发前，应以协议草案与当前代码为依据核对命令语义，避免将目标协议误认为已经实现。
