# 硬件物料清单（BOM）初稿

## 1. 文档范围

本文档用于协作者核对智能窗帘机器人当前设计资料中的硬件物料，为后续复现、接线、装配和测试文档提供输入。

本文档是协作核对用初稿，仅反映仓库资料中能够追溯到的设计信息，不代表物料已经采购、已经焊接或装配，也不代表整机已经通过联调、负载或安全验证。数量带有“设计位号”或来自结构文件的条目，仅表示资料中出现的设计对象；真实样机数量仍应以实物和最终装配方案为准。

本文档不记录供应商链接、价格或采购状态。资料无法确认的型号、规格、接口或数量均标记为待确认。

### 2.2 确认规则

| 当前确认状态 | 含义 |
| --- | --- |
| 资料中已确认 | 型号或物料在现有设计/结构资料中有明确文字或位号证据；不等同于已采购或已装机验证 |
| 从文件名可识别，规格待核对 | 文件名能够识别部件用途或类别，但规格、数量或适配关系仍未确认 |
| 设计需要，型号待确认 | 现有设计或复现流程需要该项，但资料不足以确定最终型号 |
| 待实物核对 | 是否存在、数量、版本或装配使用方式需要对照实物确认 |

## 7. 与后续复现文档的关系

- `docs/wiring.md` 应在 MCU、PCB 版本、蓝牙模块、电机驱动、电源方案和调试接口核对后，记录逐项接线和引脚映射。
- `docs/mechanical-assembly.md` 应以最终选用的 STL/结构件版本为基础，补充打印数量、装配顺序、紧固件、弹簧/轴承配置以及机械安全注意事项。
- `docs/test-checklist.md` 应在 BOM、接线和装配信息可复核后，覆盖上电前检查、烧录、通信、电机空载/负载、传感器、电池供电、停止保护和连续运行验证。

在上述文档和实物核对完成前，本 BOM 只能作为设计资料索引与待确认清单使用，不能作为已验证的复现采购清单。
